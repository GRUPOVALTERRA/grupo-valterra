4 · Decisiones arquitectónicas confirmadas

CSP gradual

Report-Only 48h post-deploy. Si dashboard Vercel no reporta violations, se mueve a enforce. Mitigación: previene incidents por whitelist incompleta.

Sentry sin tracing

tracesSampleRate: 0. Solo errores. Cubre 5k events/mes free tier sin riesgo de quema. Sprint 11+ evalúa habilitar perf monitoring si vale el costo.

Resend sandbox

Sender onboarding@resend.dev hasta tener valterra.com.ar verificado. Cambio futuro vía env RESEND_SENDER sin redeploy.

Multi-email future-proof

NOTIFICATION_EMAIL ya parsea CSV. Hoy 1 email; agregar más es solo editar la env. Tags Resend para analytics por tipo de evento.

agency_id canónico

Ambas tablas (leads y properties) usan el mismo nombre. Sprint 10 crea FK contra agencies sin debate.

Sourcemaps deferred

Sentry captura igual sin SENTRY_AUTH_TOKEN. Stack traces vienen minificados pero legibles. Activar symbolication en Sprint 11.

7 · Deploy checklist (resumen)

Detalle paso a paso en docs/SPRINT-9.5-DEPLOY-CHECKLIST.md.

  • 0. Pre-flight: npm install, npm run typecheck, npm run lint local
  • 1. Sentry: crear cuenta + proyecto Next.js + copiar DSN a .env.local
  • 2. Resend: crear cuenta + API key + setear NOTIFICATION_EMAIL
  • 3. Smoke test local: enviar lead de prueba → ver email + Sentry sin errores
  • 4. Migration 0003: aplicar en Supabase Studio + verificar columna renombrada
  • 5. Env vars Vercel: SENTRY_DSN + NEXT_PUBLIC_SENTRY_DSN + RESEND_API_KEY (Sensitive) + NOTIFICATION_EMAIL
  • 6. Commit + push: un solo commit A+B con el mensaje arriba
  • 7. Validar deploy: build Vercel verde
  • 8. Post-deploy: /api/health con commit+region+deploy_id+sentry active · CSP headers visibles en Network tab · share preview en opengraph.dev · lead test e2e
  • 9. 48h post-deploy: switch CSP Report-Only → enforce si no hubo violations
  • 10. Rollback: documentado (Vercel promote previous, SQL rollback, env unset)
